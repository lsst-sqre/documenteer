### New features

- The bundled `documenteer.ext.diagrams` extension can now render diagrams as SVG. Set the new `diagrams_output_format` configuration value to `"svg"` in your `conf.py` for scalable vector output in HTML; the default remains `"png"`. SVG output is self-contained because the provider node icons (which the `diagrams` package references by absolute filesystem path) are embedded as base64 `data:` URIs, so diagrams render correctly once the documentation is deployed. LaTeX/PDF builds always use PNG, since `pdflatex` cannot embed SVG images.
