### New features

- Added a `documenteer technote validate` command that checks a technote's metadata before a build. This first release verifies that `technote.toml` conforms to the technote schema (`TN001`) and that every author declares a resolvable `internal_id` in the Rubin author database — flagging authors missing an ID (`TN101`), IDs absent from the database (`TN102`), and an unreachable database (`TN103`, a warning). Each finding is prefixed with a stable linter-style code, and `--strict/-s` promotes warnings to errors so CI can fail on them.
