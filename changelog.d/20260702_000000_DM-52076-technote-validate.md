### New features

- Added a `documenteer technote validate` command that checks a technote's metadata and structure before a build. It verifies that `technote.toml` conforms to the technote schema (`TN001`) and that `requirements.txt` declares `documenteer[technote]` (`TN002`) without pinning Sphinx separately (`TN003`); that every author declares a resolvable `internal_id` in the Rubin author database — flagging authors missing an ID (`TN101`), IDs absent from the database (`TN102`), and an unreachable database (`TN103`, a warning); and that the content declares a non-empty abstract via the `.. abstract::` directive (`TN201`) rather than an ordinary section heading (`TN202`). Each finding is prefixed with a stable linter-style code, and `--strict/-s` promotes warnings to errors so CI can fail on them.
