### Other changes

- The `test`, `typing`, and `demo` nox sessions are now parametrized over both Python (3.12, 3.13) and Sphinx (8, 9, dev), so their session IDs carry both factors (e.g. `test-3.13(sphinx='8')`). A bare `nox` invocation still runs a lean default set (Python 3.13 × Sphinx 8/9), while `nox -s test` runs the full grid. uv furnishes the requested interpreter on demand.
- CI now collects coverage from every cell of the Python × Sphinx matrix and combines it into a single report posted to the GitHub job summary. Each matrix cell sets `DOCUMENTEER_COVERAGE_NO_COMBINE` to upload its raw `.coverage.*` data, which a central `coverage-combine` job merges once. A `test-matrix` roll-up status check now gates on the whole matrix so the branch protection ruleset references one check instead of every cell, and the `demo` end-to-end build now runs as a smoke test in every cell of the matrix (folded into the `test` job).
