### New features

- The `diagrams` directive for architectural "diagrams as code" is now provided by Documenteer itself as the bundled `documenteer.ext.diagrams` extension, replacing the unmaintained `sphinx-diagrams` dependency. The vendored extension is compatible with Sphinx 8 and 9 (the last `sphinx-diagrams` release imports `sha1` from `sphinx.util`, which no longer exists in Sphinx 9). It is enabled by default in technotes; for user guides, add `"documenteer.ext.diagrams"` to your `extensions` and install the `diagrams` package plus graphviz. A thin `sphinx_diagrams` back-compat shim is retained, so existing `from sphinx_diagrams import SphinxDiagram` imports and `extensions = ["sphinx_diagrams"]` configurations keep working.

### Bug fixes

- Editing a `diagrams` diagram now re-renders it. The vendored `documenteer.ext.diagrams` names each rendered image after a hash of the diagram source (as `sphinx.ext.graphviz` does), so changing the source produces a new filename instead of serving a stale cached image. Previously the rendered PNG would not update until the build directory was wiped.
