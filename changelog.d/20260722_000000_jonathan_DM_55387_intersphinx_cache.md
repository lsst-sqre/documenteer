### New features

- Documentation builds now prefetch intersphinx object inventories from Ook's inventory cache service, so builds no longer depend on third-party site availability to resolve cross-references. The new `documenteer.ext.intersphinxcache` extension — registered by both the user-guide and technote configuration presets — runs before `sphinx.ext.intersphinx` processes its mapping: it fetches each entry's `objects.inv` from the service, writes it to the build directory, and rewrites that entry's inventory location to the local file. Target URIs are left unchanged, so resolved links still point at the upstream sites. Prefetching authenticates with a bearer token from `OOK_TOKEN`; when the token is unset (fork pull requests, local builds) the extension is a complete no-op, and any per-inventory service failure (unauthorized, unreachable, server error, timeout) falls back to a direct origin fetch with a warning, so an Ook outage can never make a build worse than stock intersphinx. New `[sphinx.intersphinx_cache]` settings in `documenteer.toml` configure the integration for guides (`use_service`, `service_url`); technotes use the defaults and can override the corresponding `documenteer_intersphinx_cache_*` values in `conf.py` after the preset import.
