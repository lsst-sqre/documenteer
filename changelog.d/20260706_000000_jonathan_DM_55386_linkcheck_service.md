### New features

- The user-guide configuration now checks external links with Ook's link-check service. The new `documenteer.ext.linkcheckservice` extension registers a builder over Sphinx's built-in `linkcheck` name, so existing `make linkcheck` invocations transparently submit the project's external links (collected from the resolved doctrees, with `linkcheck_ignore` patterns applied client-side) to the service and poll for its cached, retry-buffered results. The submission authenticates with a bearer token from the `OOK_TOKEN` environment variable, derives the LSST the Docs project slug from the subdomain of `project.base_url`, and flags default-branch builds from the GitHub Actions environment (overridable with `DOCUMENTEER_LINKCHECK_DEFAULT_BRANCH`). New `[sphinx.linkcheck]` settings in `documenteer.toml` configure the integration: `use_service`, `service_url`, `poll_budget`, `strict`, and `slug`.
