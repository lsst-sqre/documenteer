### New features

- The user-guide configuration now checks external links with Ook's link-check service. The new `documenteer.ext.linkcheckservice` extension registers a builder over Sphinx's `linkcheck` name, so existing `make linkcheck` invocations transparently submit the project's external links to the service and report its cached, retry-buffered results, writing a machine-readable `linkcheck.json` artifact to the build output. Submissions authenticate with a bearer token from `OOK_TOKEN`; `broken` links fail the build, while `redirected`, `failing`, `unsupported`, and `blocked` links are reported at info level so a `-W` build doesn't fail on them (matching Sphinx's built-in linkcheck). New `[sphinx.linkcheck]` settings in `documenteer.toml` configure the integration (`use_service`, `service_url`, `poll_budget`, `strict`, `origin_base_url`). When `OOK_TOKEN` is missing or rejected the builder falls back to Sphinx's built-in in-process `linkcheck`; when the service is unreachable or the poll budget is exhausted it warns and exits 0 by default, or fails the build if `strict = true`. Set `use_service = false` to always use the built-in builder.
- The technote configuration also checks external links with the service, deriving the origin base URL from the technote's canonical URL in `technote.toml` (falling back to `https://<handle>.lsst.io`). Every setting is overridable in the technote's `conf.py` after the preset import (for example `documenteer_linkcheck_use_service = False`), and the technote `tox.ini` template now passes `OOK_TOKEN` through, so a technote whose CI sets the token opts in and one without a token falls back to the built-in `linkcheck`.
