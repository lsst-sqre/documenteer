### New features

- The user-guide configuration now checks external links with Ook's link-check service. The new `documenteer.ext.linkcheckservice` extension registers a builder over Sphinx's built-in `linkcheck` name, so existing `make linkcheck` invocations transparently submit the project's external links (collected from the resolved doctrees, with `linkcheck_ignore` patterns applied client-side) to the service and poll for its cached, retry-buffered results. The submission authenticates with a bearer token from the `OOK_TOKEN` environment variable, derives the LSST the Docs project slug from the subdomain of `project.base_url`, and flags default-branch builds from the GitHub Actions environment (overridable with `DOCUMENTEER_LINKCHECK_DEFAULT_BRANCH`). New `[sphinx.linkcheck]` settings in `documenteer.toml` configure the integration: `use_service`, `service_url`, `poll_budget`, `strict`, and `slug`. The builder reports summary counts by status and lists each link needing attention with its page, HTTP status, and redirect location; `broken` links fail the build with a nonzero exit status while `redirected`, `failing` (retries pending), and `unsupported` links produce warnings only. A machine-readable `linkcheck.json` artifact with the full per-URL results is written to the build output directory.
