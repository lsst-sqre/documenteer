### New features

- The user-guide configuration now checks external links with Ook's link-check service. The new `documenteer.ext.linkcheckservice` extension registers a builder over Sphinx's built-in `linkcheck` name, so existing `make linkcheck` invocations transparently submit the project's external links (collected from the resolved doctrees, with `linkcheck_ignore` patterns applied client-side and the non-checkable URIs the built-in builder never checks — empty, `#fragment`, `mailto:`, `tel:`, and non-http(s) schemes — filtered out) to the service and poll for its cached, retry-buffered results. The submission authenticates with a bearer token from the `OOK_TOKEN` environment variable, identifies the website by its origin base URL derived from `project.base_url`, and flags builds of the origin's default version from the GitHub Actions environment (overridable with `DOCUMENTEER_LINKCHECK_DEFAULT_BRANCH`). A submission whose URLs all resolve immediately is reported straight from the submission response; otherwise the builder polls the service until the check completes. New `[sphinx.linkcheck]` settings in `documenteer.toml` configure the integration: `use_service`, `service_url`, `poll_budget`, `strict`, and `origin_base_url`. The builder reports summary counts by status and lists each link needing attention with its page, HTTP status, and redirect location; `broken` links fail the build with a nonzero exit status while `redirected`, `failing` (retries pending), and `unsupported` links are reported at info level, so a warnings-as-errors (`-W`) build does not fail when the check finds nothing broken (matching Sphinx's built-in linkcheck, which fails via exit status rather than warnings). Each URL is submitted — and annotated in the `linkcheck.json` artifact — with the complete, deduplicated, sorted set of pages that reference it, so the service records accurate origin paths for URLs that appear on more than one page. A machine-readable `linkcheck.json` artifact with the full per-URL results is written to the build output directory. Service problems degrade gracefully by default: when the service is unreachable, the `OOK_TOKEN` is missing (as in a fork's pull request build), or the polling budget is exhausted, the builder warns that the link check was skipped and the build exits 0. Setting `[sphinx.linkcheck] strict = true` makes those same conditions fail the build instead, and `[sphinx.linkcheck] use_service = false` restores Sphinx's built-in in-process `linkcheck` builder.
- The technote configuration also checks external links with Ook's link-check service. The technote preset registers `documenteer.ext.linkcheckservice` and derives the origin base URL from the technote's canonical URL in `technote.toml` (falling back to the technote handle as `https://<handle>.lsst.io`), and matches default-version detection against `[technote] github_default_branch`. Technotes don't use `documenteer.toml`, so the other service settings keep their defaults and every setting is overridable in the technote's `conf.py` after the preset import — for example, `documenteer_linkcheck_use_service = False` restores Sphinx's built-in `linkcheck` builder.
