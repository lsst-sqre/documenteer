### Other changes

- Modernized the frontend asset toolchain to Node 24 LTS (`.nvmrc` now pins `v24.18.0`) and refreshed the webpack build dependencies to their current major releases (`copy-webpack-plugin` 14, `css-loader` 7, `css-minimizer-webpack-plugin` 8, `postcss-loader` 8, `sass-loader` 17, `style-loader` 4, `webpack-cli` 7) along with `sass` and `prettier`. The bundled `@lsst-sqre/rubin-style-dictionary` design tokens are updated from 0.4 to 0.7.
- `sass-loader` 16 dropped the legacy Dart Sass JS API, so `webpack.config.js` now pins the `sass` (Dart Sass) implementation and opts into the shared `modern-compiler` API explicitly.
- The CI asset workflows now install from the lockfile with `npm ci` (instead of `npm install`) for reproducible builds, and the GitHub Actions are bumped to versions that run on Node 24 (`setup-uv` v8.2.0, `upload-artifact` v7, `download-artifact` v8).
- CI now uploads the rendered demo technotes as build artifacts so they can be reviewed from the Actions UI. The PR workflow uploads the canonical cell (Python 3.13 × current stable Sphinx) with a 7-day retention, while the periodic `ci-cron` workflow also uploads the Sphinx dev build and keeps both for 60 days as a baseline for spotting dev-build regressions.
